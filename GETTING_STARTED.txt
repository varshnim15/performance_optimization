LOCAL DEVELOPMENT (Docker Compose):
-----------------------------------
1. cd Performance_Optimization
2. docker-compose up -d
3. Open http://localhost:3000 (Dashboard)
   
Services:
  • Frontend: http://localhost:3000
  • Backend: http://localhost:8080/api
  • AI Service: http://localhost:8001/docs (Swagger)
  • Grafana: http://localhost:3001 (admin/admin)
  • Prometheus: http://localhost:9090

KUBERNETES DEPLOYMENT:
---------------------
1. cd infra/kubernetes
2. kubectl apply -f namespace.yaml
3. kubectl apply -f -R .

================================================================================
🆘 SUPPORT & TROUBLESHOOTING
================================================================================

Common Issues:

Q: Services won't start?
A: docker-compose down -v && docker-compose build --no-cache && docker-compose up

Q: Database connection error?
A: Check MySQL is running: docker logs perf-optimizer-mysql

Q: Frontend not connecting to backend?
A: Verify NEXT_PUBLIC_API_URL environment variable

Q: Kubernetes pods not starting?
A: kubectl describe pod <pod-name> -n perf-optimizer

For detailed troubleshooting:
  • See QUICK_START.md → Troubleshooting section
  • Check docker-compose logs: docker-compose logs -f
  • Check K8s logs: kubectl logs <pod> -n perf-optimizer
